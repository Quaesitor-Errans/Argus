# Storage

## Назначение

Storage отвечает за взаимодействие с базой данных.

Он:

- сохраняет данные;
- получает данные;
- изменяет данные;
- удаляет данные;
- предоставляет специализированные запросы через репозитории.

Storage не:

- скачивает документы;
- анализирует содержимое;
- управляет RSS;
- определяет порядок выполнения pipeline.

## Управление сессиями

Сервисы получают SQLAlchemy-сессии через `DatabaseSessionManager`.

Менеджер отвечает за:

- создание отдельной сессии для операции сервиса;
- передачу сессии сервису;
- откат незавершённой транзакции при необработанном исключении;
- гарантированное закрытие сессии.

Пример использования:

```python
with session_manager.session() as session:
    repository = ArticleRepository(session)
