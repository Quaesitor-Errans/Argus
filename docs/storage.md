# Storage

## Назначение

Storage отвечает за взаимодействие с базой данных.

Он:

- сохраняет данные;
- получает данные;
- изменяет данные;
- удаляет данные;
- предоставляет специализированные запросы через репозитории.

Storage не:

- скачивает документы;
- анализирует содержимое;
- управляет RSS;
- определяет порядок выполнения pipeline.

## Управление сессиями

Сервисы получают SQLAlchemy-сессии через `DatabaseSessionManager`.

Менеджер отвечает за:

- создание отдельной сессии для операции сервиса;
- передачу сессии сервису;
- откат незавершённой транзакции при необработанном исключении;
- гарантированное закрытие сессии.

Пример использования:

```python
with session_manager.session() as session:
    repository = ArticleRepository(session)
```

На текущем этапе репозитории по-прежнему явно фиксируют изменения через
`commit()`.

`DatabaseSessionManager` намеренно не выполняет автоматический `commit`,
поскольку существующие репозитории пока самостоятельно определяют границы
транзакций. Изменение этой модели должно выполняться отдельно и сопровождаться
тестами транзакционного поведения.

## Обработка ошибок

Если исключение покидает управляемый контекст, менеджер:

1. вызывает `rollback()`;
2. повторно выбрасывает исходное исключение;
3. закрывает сессию.

Сервисы могут самостоятельно обработать ошибку отдельного элемента pipeline,
выполнить локальный `rollback()` и продолжить обработку остальных элементов.

## Репозитории

Все специализированные репозитории наследуют типизированный
`BaseRepository`.

Базовый репозиторий предоставляет только общие операции хранения:

- получение модели по первичному ключу;
- добавление одной модели;
- добавление нескольких моделей;
- удаление модели;
- отправку накопленных изменений через `flush()`;
- обновление состояния модели через `refresh()`.

Пример объявления специализированного репозитория:

```python
class ArticleRepository(BaseRepository[Article]):
    def __init__(self, session: Session) -> None:
        super().__init__(
            session=session,
            model_type=Article,
        )
```

`BaseRepository` намеренно не вызывает `commit()` или `rollback()`.

Скрытые транзакционные действия в базовом классе сделали бы невозможными
атомарные операции, затрагивающие несколько репозиториев. Границы транзакций
должны оставаться явными.

## Специализированные запросы

Запросы, относящиеся к конкретной модели или предметной области, остаются в
специализированных репозиториях.

Например:

- поиск статьи по URL находится в `ArticleRepository`;
- выборка статей для анализа находится в `DiscourseAnalysisRepository`;
- переходы состояния обработки находятся в `ProcessingStateRepository`.

`BaseRepository` не должен превращаться в универсальный слой, содержащий
предметную логику.

Все новые запросы должны использовать SQLAlchemy 2 `select()` API.
Legacy-интерфейс `session.query()` в Argus не используется.
